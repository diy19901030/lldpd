Detailed Description

Error codes and logs handling.

When a function returns a pointer, it may return NULL to indicate an error condition. In this case, it is possible to use lldpctl_last_error() to get the related error code which is one of the values in lldpctl_error_t enumeration. For display purpose lldpctl_strerror() may be used to translate this error code.

When a function returns an integer, it may return a negative value. It usually means this is an error but some functions may return a legetimate negative value (for example lldpctl_atom_get_int()). When there is a doubt, lldpctl_last_error() should be checked.

An error is attached to a connection. If there is no connection, no error handling is available. Most functions use a connection or an atom as first argument and therefore are attached to a connection. To get the connection related to an atom, use lldpctl_atom_get_connection().

Also have a look at lldpctl_log_callback() function if you want a custom log handling.

Enumeration Type Documentation

enum lldpctl_error_t

Possible error codes for functions that return negative integers on this purpose or for lldpctl_last_error().

Enumerator:
LLDPCTL_NO_ERROR 

No error has happened (yet).

LLDPCTL_ERR_WOULDBLOCK 

A IO related operation would block if performed.

LLDPCTL_ERR_EOF 

A IO related operation has reached a end of file condition.

LLDPCTL_ERR_NOT_EXIST 

The requested information does not exist. For example, when requesting an inexistant information from an atom.

LLDPCTL_ERR_CANNOT_CONNECT 

Cannot connect to the lldpd daemon. This error only happens with default synchronous handlers.

LLDPCTL_ERR_INCORRECT_ATOM_TYPE 

Atom is of incorrect type for the requested operation.

LLDPCTL_ERR_SERIALIZATION 

An error occurred during serialization of message.

LLDPCTL_ERR_INVALID_STATE 

The requested operation cannot be performed because we have another operation already running.

LLDPCTL_ERR_CANNOT_ITERATE 

The provided atom cannot be iterated.

LLDPCTL_ERR_BAD_VALUE 

The provided value is invalid.

LLDPCTL_ERR_CANNOT_CREATE 

No new element can be created for this element.

LLDPCTL_ERR_FATAL 

The library is under unexpected conditions and cannot process any further data reliably.

LLDPCTL_ERR_NOMEM 

Out of memory condition. Things may get havoc here but we should be able to recover.

LLDPCTL_ERR_CALLBACK_FAILURE 

An error occurred in a user provided callback.

Function Documentation

lldpctl_error_t lldpctl_last_error ( lldpctl_conn_t *  conn)

Get the last error associated to a connection to lldpd.

Parameters:
connPreviously allocated handler to a connection to lldpd.
Returns:
0 if no error is currently registered. A negative integer otherwise.

For functions returning int, this function will return the same error number. For functions returning something else, you can use this function to get the appropriate error number.

void lldpctl_log_callback ( void(*)(int severity, const char *msg)  cb)

Setup log handlers.

By default, liblldpctl will log to stderr. The following function will register another callback for this purpose. Messages logged through this callback may be cryptic. They are targeted for the developer. Message for end users should rely on return codes.
