Detailed Description

Special access functions.

Most information can be retrieved through lldpctl_atom_get(), lldpctl_atom_get_int(), lldpctl_atom_get_str() or lldpctl_atom_get_buffer() but some information can only be retrieved through special functions because IO operation is needed (and also, for some of them, because we don't have an atom yet).

lldpctl_atom_t* lldpctl_get_configuration ( lldpctl_conn_t *  conn)

Retrieve global configuration of lldpd daemon.

Parameters:
connConnection with lldpd.
Returns:
The global configuration or NULL if an error happened.

This function will make IO with the daemon to get the configuration. Depending on the IO model, information may not be available right now and the function should be called again later. If NULL is returned, check the last error. If it is LLDPCTL_ERR_WOULDBLOCK, try again later.

lldpctl_atom_t* lldpctl_get_interfaces ( lldpctl_conn_t *  conn)

Retrieve the list of available interfaces.

Parameters:
connPreviously allocated handler to a connection to lldpd.
Returns:
The list of available ports or NULL if an error happened.

This function will make IO with the daemon to get the list of ports. Depending on the IO model, information may not be available right now and the function should be called again later. If NULL is returned, check what the last error is. If it is LLDPCTL_ERR_WOULDBLOCK, try again later (when more data is available).

The list of available ports can be iterated with lldpctl_atom_foreach().

lldpctl_atom_t* lldpctl_get_port ( lldpctl_atom_t *  port)

Retrieve the information related to a given interface.

Parameters:
portThe port we want to retrieve information from. This port is an atom retrieved from an interation on lldpctl_get_interfaces().
Returns:
Atom related to this port which may be used in subsequent functions.

This functions may have to do IO to get the information related to the given port. Depending on the IO mode, information may not be available tight now and the function should be called again later. If NULL is returned, check what the last error is. If it is LLDPCTL_ERR_WOULDBLOCK, try again later (when more data is available).
