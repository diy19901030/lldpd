\subsection{\-Detailed \-Description}
\-Error codes and logs handling.

\-When a function returns a pointer, it may return {\ttfamily \-N\-U\-L\-L} to indicate an error condition. \-In this case, it is possible to use \doxyref{lldpctl\-\_\-last\-\_\-error()}{p.}{group__lldpctl__errors__logs_ga591c61c3e5c5dc5ab87ec238bece4571} to get the related error code which is one of the values in \doxyref{lldpctl\-\_\-error\-\_\-t}{p.}{group__lldpctl__errors__logs_gad9b373c0bb7b1cf850f5b1901b33e822} enumeration. \-For display purpose \doxyref{lldpctl\-\_\-strerror()}{p.}{group__lldpctl__errors__logs_gab88dc1cdcca63b82e652edbca3497714} may be used to translate this error code.

\-When a function returns an integer, it may return a negative value. \-It usually means this is an error but some functions may return a legetimate negative value (for example \doxyref{lldpctl\-\_\-atom\-\_\-get\-\_\-int()}{p.}{group__lldpctl__atoms_gaef010e71841ae2fa831b066265d5c06c}). \-When there is a doubt, \doxyref{lldpctl\-\_\-last\-\_\-error()}{p.}{group__lldpctl__errors__logs_ga591c61c3e5c5dc5ab87ec238bece4571} should be checked.

\-An error is attached to a connection. \-If there is no connection, no error handling is available. \-Most functions use a connection or an atom as first argument and therefore are attached to a connection. \-To get the connection related to an atom, use \doxyref{lldpctl\-\_\-atom\-\_\-get\-\_\-connection()}{p.}{group__lldpctl__atoms_ga334affdf0d559a1850613567d98de490}.

\-Also have a look at \doxyref{lldpctl\-\_\-log\-\_\-callback()}{p.}{group__lldpctl__errors__logs_ga8554cbee2503ca1f1c0c58aad1e6744f} function if you want a custom log handling.
